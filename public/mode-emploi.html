Guide Officiel

Mode d'emploi complet

Guide a jour: denicheur modale, liste denicheur, modes de liste, detail match, coupon, exports, Telegram, limites API et pari responsable.

1. Demarrage rapide

Etape A

Ouvre la page d'accueil. Tu peux lancer le denicheur (bouton vert) pour un match ~10 min, puis filtrer une ligue et un mode: A venir, Turbo Top 10, Denicheur (liste), En cours.

Etape B

Clique Detail predictions pour voir l'analyse complete du match (moteur IA + marches + validation).

Etape C

Va sur Page Coupon, regle le nombre de matchs, la ligue et le profil de risque, puis genere.

2. Denicheur (bouton sur l'accueil)

Clique sur Lancer le denicheur (match ~10 min) sous le titre. Le site recharge les matchs puis cherche un match a venir dont le coup d'envoi tombe dans une fenetre d'environ 10 minutes (entre 5 et 15 min). Si vide, la recherche s'elargit puis on garde le meilleur score denicheur.

  • Fenetre modale: methode prudente (priorite fiabilite + cotes stables), ligue, equipes, cotes 1X2, probabilite conservatrice (marge bookmaker) et confiance prudente (pas un score agressif).
  • Memoire locale: les 3 derniers choix sont stockes sur ton navigateur pour ton suivi perso.
  • Accessibilite: Echap ferme la modale; Tab boucle dans la fenetre.
  • Recherche: le bouton est desactive pendant le chargement pour eviter les doubles clics.
  • Option complete — choisir au mieux: case a cocher sous le bouton. Filtres ultra stricts (liquidite, fiabilite elevee, aucun mouvement sur 1X2). Si rien ne convient, le denicheur n'affiche aucun match — decoche pour revenir au mode prudent.

Indicatif uniquement — reserve aux majeurs, pari responsable, aucun gain garanti.

Generation

8. Resolution des erreurs frequentes

  • Route API introuvable: redemarre le serveur puis recharge la page en dur (Ctrl+F5).
  • Erreur JSON/HTML: ancien frontend cache (service worker), fais un hard refresh ou vide le cache du site.
  • Variables serveur: copie .env.example vers .env puis renseigne les cles (Telegram, Anthropic, etc.).
  • Surveillance: le serveur expose GET /health (JSON: uptime) pour hebergement et monitoring.
  • Match deja demarre: retire le pick ou regenere en mode A venir.
  • Telegram echec: verifie TELEGRAM_BOT_TOKEN et qu'un chat existe avec le bot.
  • Trop de requetes (429): le serveur limite les envois Telegram / PDF / images par minute — patiente.
  • Bouton grise: genere d'abord un coupon non vide.

9. Bonnes pratiques de discipline

  • Ne jamais forcer un coupon si la validation demande correction.
  • Eviter les coupons trop longs; preferer qualite a quantite.
  • Surveiller les drifts forts avant envoi final.
  • Aucune combinaison n'est garantie gagnante.

10. FAQ rapide

Q: Pourquoi je ne vois pas certains matchs ?
R: Filtre ligue/mode actif, ou match pas dans le flux courant.

Q: Pourquoi un pick est remplace ?
R: Ticket Shield detecte un risque (drift, marche indisponible, faible confiance, match demarre).

Q: Je veux partager vite ?
R: Utilise Envoi Groupe 1-Clic.

Q: Le denicheur ne propose rien ?
R: Actualise les matchs; fenetre horaire vide; ou option complete activee sans match assez propre — decoche l'option ou reessaie plus tard.
